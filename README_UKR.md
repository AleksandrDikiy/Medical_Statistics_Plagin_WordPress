# Medical Statistics — WordPress Plugin

WordPress-плагін для особистого ведення медичної статистики. Дозволяє імпортувати PDF-результати аналізів із **CSD LAB**, зберігати їх у власних таблицях бази даних та переглядати у зручному двоколонковому інтерфейсі.

## Можливості

- 📄 **Імпорт PDF** — парсинг реальних PDF-результатів CSD LAB (формат ZIP-архіву з текстовими сторінками)
- 🏷️ **Автоматичне розпізнавання** показників: назва, код (укр./англ.), референтний діапазон, одиниці виміру, категорія
- 🔴 **Виділення відхилень** — значення поза нормою підсвічуються червоним
- 📋 **Список замовлень** із пошуком та фільтром за датою
- 📑 **Пагінація** (10 замовлень на сторінку)
- 🖱️ **Drag & Drop** — перетягніть PDF прямо у вікно браузера
- 🔒 **Власна роль** `medical_statistics` — доступ лише авторизованим користувачам
- ⚡ **AJAX** — всі взаємодії без перезавантаження сторінки

## Встановлення

### Варіант 1 — через wp-admin

1. Завантажте `medical-statistics.zip`
2. У WordPress-адмінці: **Плагіни → Додати новий → Завантажити плагін**
3. Оберіть ZIP-архів → **Встановити зараз** → **Активувати**

### Варіант 2 — вручну

```bash
cd /var/www/html/wp-content/plugins
unzip medical-statistics.zip
```

Потім активуйте плагін у **Плагіни → Встановлені плагіни**.

### Після активації

Плагін автоматично:
- Створює три таблиці БД (`med_indicator`, `med_ordering`, `med_measurement`)
- Реєструє роль WordPress `medical_statistics`

---

## Використання

### 1. Призначити роль користувачу

```
Користувачі → [ваш профіль] → Роль → Medical Statistics
```

### 2. Додати шорткод на сторінку

```
[medical_statistics]
```

### 3. Імпортувати PDF

Натисніть кнопку **«Імпорт з PDF»** або перетягніть PDF-файл у головну панель.  
Підтримуються файли CSD LAB у форматі ZIP-архіву (стандартний вивід лабораторії).

### 4. Переглядати результати

- Використовуйте **пошук** та **фільтр дат** у лівій панелі
- Показники **поза референтним діапазоном** відображаються червоним
- Клікайте по замовленнях у списку для перемикання

## База даних

Плагін створює три таблиці з префіксом `{wpdb_prefix}med_`:

### `med_indicator` — каталог показників

| Поле | Тип | Опис |
|------|-----|------|
| `id` | INT | Primary key |
| `name` | VARCHAR(250) | Повна назва показника |
| `code_ukr` | VARCHAR(20) | Код українською (ТТГ, ЛПВЩ…) |
| `code_eng` | VARCHAR(20) | Код англійською (TSH, HDL…) |
| `min` | DECIMAL(12,4) | Нижня межа норми |
| `max` | DECIMAL(12,4) | Верхня межа норми |
| `measure` | VARCHAR(50) | Одиниця виміру |
| `category` | VARCHAR(100) | Розділ аналізів |
| `interpretation_hint` | MEDIUMTEXT | Пояснення з PDF |

### `med_ordering` — замовлення (один рядок = один PDF)

| Поле | Тип | Опис |
|------|-----|------|
| `id` | INT | Primary key |
| `order_number` | VARCHAR(50) | Номер CSD (CS0645298) |
| `patient_name` | VARCHAR(250) | ПІБ пацієнта |
| `patient_dob` | DATE | Дата народження |
| `collection_date` | DATETIME | Дата та час забору |
| `doctor_name` | VARCHAR(250) | Лікар |
| `branch_info` | MEDIUMTEXT | Адреса забірного пункту |

### `med_measurement` — результат вимірювання

| Поле | Тип | Опис |
|------|-----|------|
| `id` | INT | Primary key |
| `id_order` | INT | FK → med_ordering (CASCADE DELETE) |
| `id_indicator` | INT | FK → med_indicator (RESTRICT) |
| `result_value` | VARCHAR(50) | Значення (зберігає оригінал: `>2000`) |
| `execution_date` | DATETIME | Дата виконання тесту |
| `is_normal` | TINYINT(1) | 1 = в нормі, 0 = відхилення |

## Формат PDF CSD LAB

Плагін розпізнає реальний формат виводу CSD LAB:

```
Пацієнт:          Іваненко Іван Іванович
Дата народження:  01.01.1985
Замовлення:       CS0645298
Дата забору:      15.03.2026 08:32
Лікар:            Петренко О.В.
Забірний пункт:   м. Київ, вул. Хрещатик 1

Гормональні дослідження

Тиреотропний гормон (ТТГ)/TSH  2.14  µIU/mL  0.27 - 4.2
15-03-2026 09:45

25-гідроксивітамін D, 25-(OH)D  12.5  ng/ml  30 - 100
15-03-2026 10:10
Значення нижче референтного діапазону свідчить про дефіцит вітаміну D.

Лейкоцити (WBC)  6.88  x 10^(9)/L  3.2 - 10.6
```

**Що розпізнається автоматично:**
- Коди у форматі `(ТТГ)/TSH`, `(ЛПВЩ)/HDLC`, `(WBC)`, `(HGB)`
- Складені одиниці: `x 10^(9)/L`, `x 10^(12)/L`, `мм/год`
- Значення з префіксом `>` або `<` (напр. `>2000`)
- Дата виконання конкретного тесту (рядок після результату)
- Назви, розбиті на два рядки (автоматичне склеювання)

## Безпека

- **Nonce-перевірка** на кожен AJAX-запит (`check_ajax_referer`)
- **Capability check** — лише роль `medical_statistics`
- **Sanitization** усіх вхідних даних (`sanitize_text_field`, `absint`, `wp_unslash`)
- **Escaping** усього виводу (`esc_html`, `esc_attr`)
- **Prepared statements** для всіх запитів до БД (`$wpdb->prepare`)
- **MIME-перевірка** через `finfo` (не лише розширення файлу)
- Завантажений PDF видаляється одразу після імпорту

## Розробка

### Локальний запуск

```bash
# Клонувати у директорію плагінів
git clone https://github.com/your-user/medical-statistics \
  wp-content/plugins/medical-statistics

# Активувати через WP CLI
wp plugin activate medical-statistics

# Призначити роль тестовому користувачу
wp user add-role <user_id> medical_statistics
```

### Якщо CSD змінить формат PDF

Встановіть `spatie/pdf-to-text` як резервний парсер:

```bash
cd wp-content/plugins/medical-statistics
composer require spatie/pdf-to-text
```

Переконайтеся, що бінарник `pdftotext` доступний у системі:
```bash
apt-get install poppler-utils   # Ubuntu/Debian
brew install poppler            # macOS
```

### AJAX Actions

| Action | Метод | Параметри |
|--------|-------|-----------|
| `med_stat_get_orders` | POST | `page`, `date_from`, `date_to`, `search` |
| `med_stat_get_order` | POST | `order_id` |
| `med_stat_import_pdf` | POST | `pdf_file` (multipart) |

## Відомі обмеження

- Плагін призначений для **особистого використання** (один пацієнт = один WP-користувач із роллю `medical_statistics`). Для мульти-пацієнтського сценарію потрібна доопрацювання схеми БД.
- Парсер орієнтований на формат **CSD LAB** станом на 2025–2026 рр. Якщо лабораторія змінить структуру PDF — може знадобитись оновлення регулярних виразів у `class-import-csd.php`.
- Повторний імпорт того самого замовлення (за `order_number`) **ігнорується** — дублікатів не створюється.
